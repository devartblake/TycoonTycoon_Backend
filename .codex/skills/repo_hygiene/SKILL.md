# Skill: Repo Hygiene

Use this skill for solution structure, package versions, docs drift, dead code, naming, and cleanup.

## Procedure

1. Identify whether cleanup is Alpha-critical.
2. Do not refactor unrelated code while implementing features.
3. Keep central package management consistent.
4. Update `.slnx` only when project membership changes.
5. Flag deprecated projects instead of deleting them unless authorized.
6. Keep docs aligned with actual code and Docker behavior.
