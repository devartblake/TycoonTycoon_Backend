# Codex Operating Guide

You are working in the Synaptix / TycoonTycoon_Backend repository.

## Mission

Execute backend implementation tasks safely and quickly for Alpha/Beta before June 1 while preserving long-term platform architecture.

## Operating rules

- Read before writing.
- Make surgical changes.
- Do not invent missing requirements.
- Prefer tests and verification over broad refactors.
- Keep feature creep out of Alpha.
- Use feature flags for incomplete non-essential systems.
- Keep local Docker as the official integration target.
- Preserve Clean Architecture boundaries.
- Never commit secrets.

## Required task report

Every completed task must include:

1. What changed.
2. Why it changed.
3. Files touched.
4. Verification commands run.
5. Any skipped or failed checks.
6. Alpha/Beta impact.
7. Long-term architecture impact.
