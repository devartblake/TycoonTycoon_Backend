# Codex PR Review Checklist

- [ ] Change is scoped to the task.
- [ ] Priority classification is included.
- [ ] Architecture layers are respected.
- [ ] No secrets or credentials are committed.
- [ ] DB changes are migration-safe.
- [ ] Endpoint contracts are tested.
- [ ] Docker/local integration impact is considered.
- [ ] Feature flag added if Alpha scope is uncertain.
- [ ] Verification commands are documented.
- [ ] Remaining risks are stated.
