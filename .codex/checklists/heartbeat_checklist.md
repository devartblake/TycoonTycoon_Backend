# Heartbeat Checklist

Before Codex starts:

- [ ] Task has a title.
- [ ] Priority is classified.
- [ ] Affected context is identified.
- [ ] `current-task.md` is updated.

During implementation:

- [ ] Planned files are listed.
- [ ] Actual changed files are listed.
- [ ] Current step is updated.
- [ ] Blockers are documented.

Before completion:

- [ ] Verification is recorded.
- [ ] `verification-log.md` is updated.
- [ ] `alpha-status.md` is updated if release readiness changed.
- [ ] Post-Alpha work is deferred explicitly.
- [ ] Final task report is produced.
